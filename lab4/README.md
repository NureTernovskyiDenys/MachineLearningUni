## 📖 Про проєкт
Цей веб-додаток створено в межах лабораторної роботи для демонстрації принципів роботи **Мережі Хопфілда** як систем асоціативної пам'яті. Додаток дозволяє мережі "запам'ятовувати" 10 чорно-білих еталонів (цифри від 0 до 9) на сітці розміром 10x10 пікселів та відновлювати їх при внесенні випадкового шуму (спотворенні).

> **Мета:** Дослідити ємність мережі та її здатність до відновлення зашумлених даних при використанні двох різних алгоритмів навчання: **Правила Хебба** та **Методу Проєкцій**.

## ✨ Основні можливості
* 🎨 **Інтерактивна сітка 10x10:** Візуалізація та ручне керування станом нейронів (+1 / -1).
* 👨‍🏫 **Два алгоритми навчання:** Класичне правило Хебба (з налаштуванням коефіцієнта навчання $\eta$) та Метод Проєкцій (Псевдоінверсія).
* 🌪 **Механізм спотворення:** Можливість додавати шум від 0 до 100% для тестування мережі.
* 🎥 **Асинхронна анімація:** Плавне покрокове оновлення стану мережі до повної стабілізації (або знаходження хибного "атрактора").
* 📊 **Модуль Benchmark (Дослідження):** Автоматичний прогін 100 тестів для різних рівнів шуму (10-50%) з розрахунком відсотка успішних відновлень.

## ⚙️ Опис налаштувань та елементів керування

| Елемент керування | Опис | Аргументи / Значення |
| :--- | :--- | :--- |
| **Таблиця з пікселями** | Сітка 10x10, де кожна клітинка — це нейрон. Відображає процес розпізнавання. | `+1` (Синій) / `-1` (Пустий) |
| **Правило навчання** | Алгоритм для розрахунку матриці вагових коефіцієнтів $W$. | `Хебба`, `Проєкцій (Псевдоінверсія)` |
| **Коефіцієнт навчання ($\eta$)** | Визначає силу кожного кроку навчання для правила Хебба. | Дробове число `0.1 – 5.0` (за замовчуванням `1.0`) |
| **Вибір Еталону** | Кнопки для швидкого завантаження 10 "ідеальних" цифрових образів. | Цифри від `0` до `9` |
| **Шум (Спотворення %)** | Інструмент для випадкової інверсії заданого відсотка пікселів. | `0% - 100%` |
| **Дослідження (Benchmark)** | Автоматизований стрес-тест стійкості мережі для збору статистики (100 прогонів). | Тести на шумах: `10%, 20%, 30%, 40%, 50%`. Результат: *Точність відновлення (0-100%)* |

## 🚀 Встановлення та запуск

Для локального запуску знадобляться встановлені [Node.js](https://nodejs.org/) (версія 18+) та Npm.

1. **Клонуйте репозиторій або перейдіть до папки:**
   ```bash
   cd ./Project-Folder
   ```

2. **Встановіть залежності:**
   ```bash
   npm install
   ```

3. **Запустіть сервер для розробки:**
   ```bash
   npm run dev
   ```

4. **Відкрийте додаток:**
   Перейдіть у браузері за адресою [http://localhost:5173/](http://localhost:5173/) (або за тією, яка вказана у терміналі).

## 📝 Покрокова інструкція для тестування (ЛР)

1. Увімкніть додаток та виберіть **Правило навчання** (почніть з Хебба).
2. Натисніть кнопку **"Навчити Мережу"**.
3. Завантажте будь-яку цифру (наприклад, `3`) через блок **"Вибір Еталону"**.
4. Встановіть повзунок **Шум** на `20%` і натисніть **"Спотворити"**. Картинка частково зруйнується.
5. Натисніть **"Розпізнати"** і спостерігайте, як мережа спробує відновити цифру.
6. Для збору даних до лабораторної: натисніть **"Run Benchmark"** і зачекайте кілька секунд. Запишіть результати точності для різних відсотків шуму. Повторіть це для правила **Проєкцій** і порівняйте ефективність.
