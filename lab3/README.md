# 🎧 Аналізатор настрою аудіозаписів (Audio Sentiment Analyzer)

Цей проєкт — це Full-Stack веб-застосунок (React + Node.js), який дозволяє завантажити аудіофайл, перетворити його на текст (Speech-to-Text) та миттєво оцінити емоційну тональність цього тексту (Sentiment Analysis).

## 🌟 Основні можливості
- **Конвертація аудіо**: Автоматичне перетворення завантаженого аудіо (WAV, MP3, FLAC) у необхідний формат (16kHz, Mono) за допомогою внутрішньої інтеграції `ffmpeg` (без ручного встановлення на комп'ютер).
- **Транскрибування (Speech-to-Text)**: Розпізнавання мовлення довготривалих аудіозаписів за допомогою Google Cloud Speech-to-Text API з підтримкою української (`uk-UA`) та англійської (`en-US`) мов.
- **Аналіз Тону (Natural Language)**: Оцінка загального емоційного забарвлення розпізнаного тексту (позитивний, негативний, нейтральний) та аналіз сили емоцій (Magnitude).
- **Візуалізація**: Сучасний та інтуїтивно зрозумілий UI з відображенням індикатора завантаження в реальному часі (через Server-Sent Events) та кольоровою підсвіткою настрою окремих речень.

## ⚙️ Технологічний стек
- **Frontend**: React, Vite, Tailwind CSS.
- **Backend**: Node.js, Express, Multer.
- **Хмарні сервіси**: Google Cloud Storage, Google Cloud Speech-to-Text, Google Cloud Natural Language.

## 📋 Вимоги (Prerequisites)
1. Встановлений Node.js (v16 або новіший).
2. Обліковий запис Google Cloud із увімкненим білінгом.
3. Увімкнені Google Cloud API:
   - **Cloud Speech-to-Text API**
   - **Cloud Natural Language API**
4. Сервісний ключ у форматі JSON (`Service Account Key`) з відповідними правами доступу.

## 🚀 Встановлення та запуск

Проєкт складається з бекенд та фронтенд частин, кожну з яких потрібно запустити в окремому терміналі.

### Крок 1: Налаштування Бекенду
1. Перейдіть до теки бекенду:
   ```bash
   cd backend
   ```
2. Встановіть залежності:
   ```bash
   npm install
   ```
3. Відкрийте файл `.env` (він містить ваші налаштування). Переконайтеся, що шлях до JSON-ключа вказаний коректно:
   ```env
   PORT=5000
   GOOGLE_APPLICATION_CREDENTIALS="шлях_до_вашого_ключа.json"
   GCS_BUCKET_NAME="audio-sentiment-analyzer-bucket-unique"
   ```
4. Запустіть бекенд сервер:
   ```bash
   npm run start
   ```

### Крок 2: Налаштування Фронтенду
1. Відкрийте другий термінал та перейдіть у теку фронтенду:
   ```bash
   cd frontend
   ```
2. Встановіть залежності:
   ```bash
   npm install
   ```
3. Запустіть середовище розробки:
   ```bash
   npm run dev
   ```
4. Відкрийте браузер за посиланням `http://localhost:5173`.

## 💡 Використання
1. Відкрийте застосунок в браузері.
2. Перетягніть або оберіть аудіофайл (MP3/WAV/FLAC) з англійським або українським мовленням.
3. Індикатор відобразить 4 етапи обробки: Отримання, Конвертація, Транскрибація і врешті-решт — Аналіз.
4. Ознайомтеся з кольоровими результатами, що вказують на тональність розпізнаного мовлення!
